# F5 — Sistema Visual do Chat do Agente e Integrações

> Documento de design produzido na Task 3.0d (ui-ux-pro-max). Autoridade de design
> para todas as tasks de UI das ondas 3, 5 e 6 da F5. Consultar antes de construir
> qualquer tela ou componente relacionado ao agente ou ao menu Integrações.

---

## 1. Identidade visual do projeto (base)

O Nexus Odoo usa **violet como cor primária** (var Tailwind `violet-500`/`violet-600`),
tema dark/light via CSS variables, componentes base-ui, Framer Motion para animações,
Lucide para ícones (stroke uniforme), tipografia do sistema (Geist / Inter).

Estilo canonizado: **Trust & Authority + Data-Dense Dashboard**, adaptado para chat/IA:
profissional, denso de informação, sem excessos visuais decorativos. Micro-animações
expressam relações causa-efeito; nunca decorativas puras.

## 2. Bubble flutuante (AgentBubble)

### Layout
- Posição: `fixed bottom-6 right-6 z-50`; mobile: `max-sm:bottom-5 max-sm:right-4`
- Tamanho: `h-14 w-14` (56px — acima do mínimo 44pt)
- Forma: `rounded-full`

### Visual
- Background: `bg-gradient-to-br from-violet-600 to-violet-500`
- Sombra: `shadow-lg shadow-violet-600/40`
- Ring interno: `ring-1 ring-white/15 ring-inset`
- Ícone: `Sparkles` (Lucide) `h-6 w-6 text-white strokeWidth={2.25}`
- Indicador online: bolinha `h-3.5 w-3.5 bg-emerald-400 ring-2 ring-background` no canto inferior direito

### Animações (Framer Motion)
- Entrada: `initial={{ opacity: 0, scale: 0.85, y: 8 }}` → spring `stiffness: 280, damping: 22`
- Hover: `scale: 1.06` (ignorado se `useReducedMotion()`)
- Tap: `scale: 0.94`
- Glow idle: `opacity: [0.25, 0.55, 0.25]` + `scale: [0.95, 1.1, 0.95]`, duration 2.8s loop, blur-xl
- Respeitar `prefers-reduced-motion`: glow e hover/tap desativados

### A11y
- `aria-label="Abrir o Agente"` 
- `aria-expanded={open}`
- Focus ring: `focus-visible:ring-3 focus-visible:ring-violet-400/60`

## 3. Painel de chat (ChatPanel)

### Layout e dimensões
- **Desktop**: float `fixed bottom-[calc(56px+24px+12px)] right-6`, width `420px`, height `70vh` (mín. 500px), `rounded-2xl`
- **Mobile** (`< sm`): full-screen `fixed inset-0`
- Hierarquia interna: header fixo → área de mensagens scroll → barra de input fixa

### Header
- Altura: `h-14` (56px)
- Conteúdo: ícone Sparkles + título "Agente" + badge status (online/offline) + botão X fechar + botão menu (3 dots: limpar conversa, ver em tela cheia)
- Border bottom: `border-b border-border`
- Fundo: `bg-background`

### Área de mensagens
- `flex-1 overflow-y-auto` com `overscroll-contain`
- Padding: `px-4 py-3`
- Gap entre mensagens: `space-y-3`
- Auto-scroll ao novo conteúdo: `scrollIntoView({ behavior: "smooth" })`

### Barra de input
- Layout: container `flex items-end gap-2 px-3 py-3 border-t border-border bg-background`
- Inner area (textarea container): `flex-1 rounded-xl border border-input bg-background min-h-[38px] max-h-[120px] px-3 py-2`
- Textarea: `resize-none` sem borda própria, `text-sm`, auto-resize pelo conteúdo
- Botão microfone: à esquerda do Send, só em idle (28px, muted-foreground)
- Botão Send: `h-8 w-8 rounded-lg` com gradient `from-violet-600 to-violet-500`, disabled quando vazio

### Animação de entrada/saída
- Entrada: `initial={{ opacity: 0, scale: 0.92, y: 16, transformOrigin: 'bottom right' }}` → `animate={{ opacity: 1, scale: 1, y: 0 }}`, spring `damping: 25, stiffness: 280`
- Saída: mesmas props mas `duration` = 60-70% da entrada (HIG: exit faster than enter)
- Mobile: entrada `y: "100%"` → `y: 0` (slide from bottom)

### A11y
- `role="dialog"` + `aria-modal` + `aria-labelledby`
- Esc fecha
- Foco no textarea ao abrir (timeout 80ms)

## 4. Bolhas de mensagem (AgentMessage)

### Estrutura geral
- Container: `group/msg flex w-full`
- User: `justify-end`; assistant: `justify-start`
- Balão: `relative max-w-[85%] rounded-2xl px-3.5 py-2.5 text-sm leading-relaxed`

### User
- Background: `bg-violet-600/15`
- Texto: `text-foreground`

### Assistant
- Background: `bg-muted`
- Texto: `text-foreground`

### Loading (3 dots)
- 3 dots `h-1.5 w-1.5 rounded-full bg-violet-500`
- Animação: `nexDotBounce` com delays 0s, 0.15s, 0.30s
- Texto: "Agente pensando…" em `text-muted-foreground`

### Streaming (token a token)
- O texto aparece inline conforme os tokens chegam via SSE
- Cursor piscante no final: `inline-block w-[2px] h-[1em] bg-violet-500 ml-0.5 animate-pulse`
- Removido assim que o evento `done` chegar

### Tool call
- Pill pequena: `flex items-center gap-2 rounded-full border border-border bg-background px-3 py-1 text-xs text-muted-foreground`
- Ícone `Database` (Lucide) `h-3 w-3`
- Texto: "Consultou MCP · `nome_da_tool`"

### Copy button
- Aparece no hover do balão: `opacity-0 group-hover/msg:opacity-100`
- Posição: `absolute -top-2 -right-2`
- 24×24px com ícone Copy/Check

### Markdown leve (sem dependências)
- `**bold**` → `<strong>`
- `` `inline` `` → `<code class="rounded bg-violet-600/10 px-1 py-0.5 font-mono text-[0.8em] text-violet-700 dark:text-violet-300">`
- Listas `- ` / `* ` → `<ul>`
- Parágrafos por linha em branco

## 6. Página `/agente` (tela cheia)

### Layout geral
- `flex h-[calc(100vh-64px)]` (sem o padding-top do main)
- Sidebar de conversas: `w-72 shrink-0 border-r border-border flex flex-col`
- Área de chat: `flex-1 flex flex-col`

### Sidebar de conversas
- Header: "Conversas" (label) + botão "+ Nova" (`variant="outline" size="sm"`)
- Lista de conversas: scroll vertical, cada item:
  - `flex flex-col gap-0.5 px-3 py-2.5 rounded-lg cursor-pointer transition-colors`
  - Ativo: `bg-violet-500/10 text-violet-600 dark:text-violet-300`
  - Hover: `hover:bg-muted/40`
  - Título: `text-sm font-medium truncate`
  - Timestamp: `text-xs text-muted-foreground`
- Estado vazio: "Nenhuma conversa ainda. Inicie uma nova."

### Área principal
- Header: nome da conversa ativa + timestamp última msg
- `flex-1 overflow-y-auto` para mensagens
- Input bar na base (mesmo padrão do ChatPanel, mas integrado na página)
- Padding generoso: `px-6 py-4`

## 7. Estados de streaming (indicador visual)

### Durante o streaming
1. **Status "pensando"** (antes do 1º token): loading bubble com 3 dots
2. **Streaming de tokens**: texto cresce inline + cursor piscante `animate-pulse`
3. **Tool call em andamento**: pill "Consultou MCP · nome" + spinner `animate-spin h-3 w-3` ao lado
4. **Done**: cursor some, sugestões aparecem com `animate-in fade-in duration-300`

### Evento SSE (contrato)
```
data: {"type":"status","status":"thinking"}
data: {"type":"token","delta":"Olá "}
data: {"type":"token","delta":"mundo"}
data: {"type":"done","suggestions":["Pergunta 1","Pergunta 2"]}
```

## 9. Checklist de qualidade (pré-entrega, toda task de UI)

- [ ] Sem emojis como ícones — usar Lucide exclusivamente
- [ ] `cursor-pointer` em todos os elementos clicáveis
- [ ] Hover states com transição `duration-200`
- [ ] Touch targets ≥ 44px (h-11 / h-10 mínimo para botões)
- [ ] `aria-label` em botões só-ícone
- [ ] `prefers-reduced-motion` respeitado (via `useReducedMotion()` do Framer)
- [ ] Contraste de texto ≥ 4.5:1 em light e dark
- [ ] Focus ring visível em todos os interativos
- [ ] Testado em 375px (mobile) e 1280px (desktop)
- [ ] Build sem erros de TypeScript
