## BLOCO 5, Integracao agente -> MCP externo

### T5.1 Modulo `src/lib/agent/external-mcp.ts`
- [ ] `slugifyServer(name, id)`: nome -> slug `[a-z0-9]` de **no maximo 8 chars**
  + fragmento curto do `id`, garantindo unicidade entre servidores de nome
  igual (Review #2 D2).
- [ ] `prefixedToolName(slug, toolName)`: monta `ext__<slug>__<toolName>`; se
  passar de 60 chars, trunca o nome real (o router mapeia prefixado -> real, o
  exposto so precisa ser unico e <= 64). Review #2 D2.
- [ ] `redactArgs(args)`: copia redigindo valores de chaves que casem
  `/token|secret|senha|password|key|authorization|bearer/i` para `"[redacted]"`;
  serializa limitado a ~2 KB. Usado no `argsPreview` do log (Review #2 D3).
- [ ] `openExternalMcpSessions(userId)`:
  - `prisma.externalMcpServer.findMany({ where: { enabled: true } })`.
  - Abre **todas as sessoes em paralelo** (`Promise.allSettled`; Review #2 D1).
    Por servidor: transport conforme `transport` (`http` ->
    `StreamableHTTPClientTransport` de `.../client/streamableHttp`; `sse` ->
    `SSEClientTransport` de `.../client/sse`, imports verificados na Review #1
    M3); auth `requestInit.headers[authHeader] = decrypt(authToken)` (reuso de
    `decrypt` de `@/lib/encryption`, valor cru, sem prefixo, Review #1 M4);
    `connect` + `listTools` com **timeout de ~4s** (corrida com timeout);
    estourou ou falhou -> `console.warn`, servidor pulado, run continua
    (Review #1 M3/M7, Review #2 D1).
  - Tools viram `McpTool` com nome de `prefixedToolName`; router
    `Map<prefixed, {serverId, serverName, session, realName}>`.
  - Retorna `{ tools, router, closeAll }`. `closeAll` fecha todas as sessoes com
    `Promise.allSettled`, nunca lanca (Review #2 d2).
- [ ] `callExternalTool(router, prefixedName, args, userId)`: resolve o servidor,
  mede duracao, `session.callTool(realName, args)`, grava `ExternalMcpCallLog`
  com `argsPreview = redactArgs(args)` (best-effort, try/catch, `console.error`).
  Erro de chamada -> log `outcome="error"` + devolve string de erro (nao lanca).

### T5.2 Integrar no `run-agent.ts`
- [ ] Apos `createMcpSession`, abrir
  `externalBundle = await openExternalMcpSessions(args.userId).catch(() => null)`.
- [ ] `tools` entregue ao LLM = `mcpToolsToProviderTools([...mcpTools,
  ...(externalBundle?.tools ?? [])])`.
- [ ] Loop de exec de tool: se `tc.name` comeca com `ext__` e ha
  `externalBundle`, rotear via `callExternalTool`; senao `session.callTool`. Se
  `ext__` sem rota (servidor caiu), devolver um `tool_result` de erro claro, sem
  tentar no MCP interno (Review #2 d1).
- [ ] `finally`: fechar `session` (ja faz) + `externalBundle?.closeAll()`
  (`closeAll` engole erros de close, nunca lanca).
- **Decisao registrada (Review #1 M5):** MCPs externos habilitados municiam o
  agente em **todos** os contextos (chat, WhatsApp, playground), como o MCP
  interno; capacidade de plataforma, sem scoping por usuario.
