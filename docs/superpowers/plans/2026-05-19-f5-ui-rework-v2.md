  vezes — corrigir o rótulo/descrição; deixar claro que **ali se escreve o
  prompt** (a identidade base do agente).

## Task 3.2 — Guardrails e limites
- [ ] Guardrails: **sem limite de 20** — ilimitado, criar quantos quiser.
- [ ] Limite de caracteres: comportamento e tom **500 → 1000**; cada regra de

  Botões um pouco **menores verticalmente**, com respiro da margem.

## Task 3.3 — "Modo de prompt manual"
- [ ] O "modo de prompt manual / substitui o prompt composto por texto bruto"
  não é claro para o usuário. **Decisão:** remover esse modo da UI (ou, se

## Task 3.4 — Recursos: Entrada de áudio (redesenho)
- [ ] A mensagem de estado do recurso de áudio deve **espelhar o estado real da

  configurado mas o modelo não suporta áudio → mensagem específica.
- [ ] Quando indisponível, o **toggle fica desabilitado** — ao passar o mouse,
  cursor "proibido" (`not-allowed`), não deixa clicar.
- [ ] **Decisão final do usuário sobre áudio:** o áudio NÃO depende do modelo de
  produção. A seção "Entrada de áudio" do componente Recursos **expande** e
  ganha seletores próprios de **Provedor + Modelo** (mesmo padrão rico —
  TierBadge, preço, descrição), listando **só modelos que entendem áudio**
  (recentes, ~GPT-4o em diante). Esse modelo de áudio vale para produção E
  playground. Default: `gpt-4o-mini-transcribe`.
- [ ] Descrição do áudio: **genérica**, sem citar "Whisper" nem provedor
  específico ("transcrição de mensagens de voz").

## Task 3.5 — Recursos: Entrada de imagem (novo)
- [ ] Acrescentar o recurso **"Entrada de imagem"** no mesmo padrão: expande,
  seletores Provedor + Modelo listando **só modelos que entendem imagem**.

## Task 3.6 — Controle de checkpoint (off / playground / produção)
- [ ] Os recursos (áudio, imagem) e a **base de conhecimento** deixam de ter
  um simples toggle on/off e passam a ter um **controle de 3 estados em

  - **2 — produção:** ícone **roxo**.
