# F1 — Fundação — Plano de Implementação

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development para implementar este plano task-by-task. Steps usam checkbox (`- [ ]`).

**Goal:** Construir a aplicação base do `nexus-odoo` — Next.js 16 + Prisma + Postgres + Redis + NextAuth + RBAC — com login, gestão de usuários, Docker e CI/CD.

**Architecture:** Monorepo Next.js (App Router). A maior parte do código é **portada do projeto irmão `nexus-insights`** (`/Users/joaovitorzanini/Developer/Claude Code/Nexus AI/Projetos Internos/nexus-insights`), adaptada: nomes `nexus_insights`→`nexus_odoo`, e removendo tudo de Chatwoot/Power BI/agente-nex. Código novo é mínimo. Cada arquivo portado é revisado antes do commit.

**Tech Stack:** Next.js 16, TypeScript, Tailwind v4, base-ui, Prisma v7 (`@prisma/adapter-pg`), PostgreSQL 16, Redis 7, NextAuth v5, BullMQ, bcryptjs, Zod, Jest.

**Spec:** `docs/superpowers/specs/2026-05-15-fundacao-design.md`

**Convenção de porte:** quando um step diz "portar `<arquivo>`", significa: copiar o arquivo correspondente do `nexus-insights`, aplicar as adaptações listadas, e remover qualquer import/lógica de Chatwoot/Power BI. A fonte é a referência completa de código.
