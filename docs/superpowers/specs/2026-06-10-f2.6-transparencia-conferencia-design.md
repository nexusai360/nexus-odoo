### §3.1 Parte A , métrica `faturamentoPorCfop` (evolução ADITIVA)

`src/lib/metrics/fiscal/faturamento-por-cfop.ts` , **manter** `semCfop: {totalItens, valorProdutos}` intacto
(lido por conferência, f1-e2e, handler , review arch A7). **Adicionar**:

- **`outrasNaoEspecificadas: {totalItens: number, valorProdutos: number, valorFinalidadeVenda: number}>`** ,
  CFOP 5949/6949 (categoria `outras`); `valorFinalidadeVenda` = parcela com `finalidade_nfe='1'` (rotulada
  como "substância a confirmar", NÃO "venda").

**Mecanismo (review arch B1/B2):** duas queries `$queryRaw` com JOIN + GROUP BY no banco (finalidade vive só
no cabeçalho):
```sql
-- (1) sem_cfop por finalidade
SELECT nf.finalidade_nfe AS finalidade, COUNT(*) n, COALESCE(SUM(i.vr_produtos),0) v
FROM fato_nota_fiscal_item i JOIN fato_nota_fiscal nf ON nf.odoo_id = i.documento_id
WHERE i.entrada_saida='1' AND i.situacao_nfe='autorizada' AND i.cfop_id IS NULL
  AND <periodo> GROUP BY nf.finalidade_nfe;
-- (2) outras 5949/6949 por finalidade
SELECT (CASE WHEN i.cfop_nome LIKE '5949%' THEN '5949' ELSE '6949' END) cfop, nf.finalidade_nfe AS finalidade,
       COUNT(*) n, COALESCE(SUM(i.vr_produtos),0) v
FROM fato_nota_fiscal_item i JOIN fato_nota_fiscal nf ON nf.odoo_id = i.documento_id
WHERE i.entrada_saida='1' AND i.situacao_nfe='autorizada'
  AND (i.cfop_nome LIKE '5949%' OR i.cfop_nome LIKE '6949%') AND <periodo> GROUP BY 1, 2;
```
`<periodo>` = mesmo filtro `data_emissao` da métrica. Number(Decimal). Custo baixo (sem_cfop ~364 linhas;
5949/6949 agregado no banco).

### §3.4 Parte B , 6 checagens na conferência

Primitivas novas em `scripts/conferencia-fiscal.ts`:
```ts
function checkPct(nome, a, b, limitePct, erros) { // GATE
  const pct = b !== 0 ? Math.abs(a - b) / Math.abs(b) * 100 : 0;
  const ok = pct < limitePct; /* log + push erros se !ok */
}
function checkBandaValor(nome, valor, min, max, alertas) { // ALERTA (loga em R$)
  const ok = valor >= min && valor <= max; /* log brl(valor) + push alertas se !ok */
}
```

| ID | Tipo | O que confere | Limite/banda (real hoje) |
|---|---|---|---|
| **C1** órfãos na base de receita | **gate ==0** | itens saída-autorizada cujo `documento_id` não casa nota | == 0 (hoje 0); alerta à parte p/ órfão global (1) |
| **C2** reconciliação item×(cabeçalho−notas-sem-item) | **gate** `checkPct` | `|SUM(item) − (SUM(nota) − SUM(notas-sem-item))|` | < 0,01% (hoje ≈ R$ 0) |
| **C3** CFOP novo em `outras` | alerta | CFOPs distintos classificados `outras` além de 5949/6949 | alerta se aparecer CFOP novo com > R$ 100k/ano |
| **C4** inversões natureza×ehReceita | alerta `checkBandaValor` | (a) ehReceita & natureza não-venda; (b) !ehReceita & natureza venda. Fonte natureza = `idsNaoVenda` (`_shared/naturezas.ts`); exclui natureza NULL | (a) [R$ 1,0mi, 3,0mi]; (b) [R$ 1,5mi, 3,5mi] |
| **C5** sem_cfop por finalidade | log | decomposição (mesma fonte da Parte A) | informativo |
| **C6** notas sem item | alerta `checkBanda` | notas saída-autorizada sem item | banda [80, 130] (hoje 101) |

C1/C2 são **gate**; C3/C4/C6 **alerta**; C5 **log**. Justificativa C2≠C1 quanto a gate: C2 reconciliado contra
(cabeçalho − notas-sem-item) é invariante estrutural (≈ R$ 0), não dado volátil , gate seguro. Reusar a
mesma fórmula de reconciliação da métrica onde possível.
